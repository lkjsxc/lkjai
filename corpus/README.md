# Corpus

## Purpose

Corpus data that is intentionally part of the repository lives here. Local
scratch outputs and temporary generated data belong under `data/`.

## Contents

- [generated/](generated/): committed validated generated corpus artifacts.
- [sources/](sources/): reviewed source JSON used by corpus builders.

## Rules

- Commit only validated corpus artifacts.
- Keep stale or quarantined generated data out of the repository.
- Preserve source license and provenance metadata.
